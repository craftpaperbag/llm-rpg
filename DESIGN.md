# 100歩の黙示録 — 設計書

> 隕石衝突まで残り100歩。現代日本で最後の選択をする、モノクロのブラウザRPG。

---

## 1. 基本仕様

| 項目 | 内容 |
|------|------|
| プラットフォーム | ブラウザ (PC + スマホ) |
| 技術 | 素のHTML / CSS / JS (ESModules)、ビルド不要 |
| 公開 | GitHub Pages (`main` ブランチ root) / ローカル `index.html` 直起動可 |
| ビジュアル | モノクロ (黒・白・グレー1階調のみ) |
| 操作 | **タップ / クリックのみ**。キーボード操作は実装しない |
| 音 | 環境音を控えめに (Web Audio APIで生成 or 短い無音圧縮素材) |
| トーン | 全編ブラックユーモア。**真エンドのみシリアス・ハードSF** |
| セーブ | localStorage (進行状況 + エンディング解放履歴) |
| 言語 | 日本語のみ |

## 3. ビジュアル設計 (モノクロ + スマホ対応)

### 配色
- 背景: `#0a0a0a` (ほぼ黒)
- 文字: `#f0f0f0` (ほぼ白)
- 弱い文字 / 罫線: `#666` (中間グレー1段階のみ)
- 警告 (歩数残少): 文字を点滅させる (色は変えない)

### フォント
- `'Noto Sans JP', 'Hiragino Sans', sans-serif` (本文)
- 等幅: `'Noto Sans Mono CJK JP', monospace` (歩数カウンタ・ASCII装飾)

### レイアウト (モバイルファースト)
```
┌────────────────────────┐  ← viewport 幅 100%、最大 480px センター寄せ
│ ☄  73 / 100   ▓▓▓▓░░░░ │  ヘッダー固定 (歩数バー)
├────────────────────────┤
│                        │
│ [駅前ロータリー]       │  シーン名
│                        │
│ 人がまばらに座り込ん   │  本文 (最大4-6行、句点で改行整形)
│ でいる。空には赤い光   │
│ の筋が走っている。     │
│                        │
├────────────────────────┤
│ ▸ 話しかける    -3歩  │  選択肢: タップ領域 高さ56px以上
│ ▸ 移動する      -5歩  │  (Apple HIG 推奨44px超)
│ ▸ 空を見る      -1歩  │
├────────────────────────┤
│ 🜲 缶コーヒー / 写真   │  持ち物 (横スクロール可)
└────────────────────────┘
```

### 演出
- 歩数が減るほど画面右上の **隕石の影** (SVG円) が拡大
- 残り20歩以下で背景にノイズオーバーレイが薄く出る
- 残り10歩以下で全体が微振動 (`@keyframes shake`)
- 場面遷移は 200ms フェード

### スマホ最適化
- `<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">`
- `touch-action: manipulation` でダブルタップズーム抑制
- 選択肢ボタンはタップ領域 56px 以上、間隔 12px
- フォントは `font-size: clamp(15px, 4vw, 18px)`
- セーフエリア対応 (`env(safe-area-inset-*)`)
- 横画面でも崩れない (max-width: 480px センター)

## 4. ゲームメカニクス

### 4.1 歩数システム
- 初期値 **100歩**
- 各選択肢に `cost: 1〜10` を設定
- `0` 以下になった瞬間、強制的にエンディング判定 → 隕石落下演出
- 移動コスト: 隣接ノード間 3〜5歩、長距離 7〜10歩

### 4.2 シーン (10ノード)

| ID | 場所 | 概要 |
|----|------|------|
| `apartment` | 自宅アパート | スタート地点。ベランダ・冷蔵庫・PCを調べられる |
| `convenience` | コンビニ | 店員、強盗、おでん。ブラックユーモアの宝庫 |
| `station` | 駅前ロータリー | 演説する人、ホームレス、迷子の犬 |
| `school` | 母校 | 元同級生 (恋人候補) と再会する可能性 |
| `office` | 職場ビル | 部長がまだ会議をしている |
| `shrine` | 神社 | 老神主が瞑想を勧めてくる |
| `riverbed` | 河川敷 | バンドが最後のライブをしている |
| `hospital` | 総合病院 | 看護師が走り回っている。倫理的選択 |
| `rooftop` | 高層ビル屋上 | 空が一番よく見える。真エンド分岐点 |
| `parents` | 実家 | 家族エンドのトリガー |

ノード間の接続図 (隣接 = 移動コスト3、それ以外 = 7):
```
parents — apartment — convenience — station — school
                          |             |       |
                       riverbed      shrine   office
                                              |
                                          rooftop
                                              |
                                          hospital
```

### 4.3 隠しパラメータ (UI非表示)

| 名前 | 増加条件 | 関連エンド |
|------|---------|-----------|
| `bond` (絆) | NPCと会話、人助け | 家族・恋人・見知らぬ人 |
| `nihil` (諦観) | 一人で過ごす、瞑想 | 達観 |
| `wrath` (怒り) | 略奪、暴力、罵倒 | 狂気 |
| `hope` (希望) | 救助、寄付、前向きな選択 | 見知らぬ人 |
| `truth` (真実) | 隕石・科学関連を調べる | 真エンド |

各エンドは複数パラメータの閾値 + 特定フラグの組み合わせで成立。

### 4.4 アイテム (例)

| ID | 名前 | 入手場所 | 用途 |
|----|------|---------|------|
| `coffee` | 缶コーヒー | コンビニ | ホームレスに渡せる |
| `photo` | 古い写真 | アパート | 実家・学校で回想トリガー |
| `omamori` | お守り | 神社 | 達観/真エンドの素材 |
| `key_home` | 実家の鍵 | アパート初期所持 | 実家解放 |
| `note` | 母からの未読LINE | 初期所持 | 開封すると `bond` +1 |
| `telescope` | 望遠鏡 | 屋上 | 真エンド条件 |
| `paper` | 研究論文の切れ端 | 病院 | 真エンド条件 |

## 5. エンディング (7種)

判定優先順位は上から (条件を最初に満たしたものが採用)。

### END 1: 真エンド「観測者」 (シリアス・ハードSF)
- **条件**: `truth >= 4` かつ `telescope` 所持 かつ `rooftop` で「空を観測する」選択
- **概要**: 主人公は望遠鏡で隕石軌道を計測し、これが自然現象ではなく**特定の周期で繰り返される観測実験**であることに気づく。地球は何者かのシミュレーションであり、滅亡は再現性検証のための初期化。最後の数秒、主人公は数式を空に書き残す。
- **トーン**: 静謐、SF、感傷を排した観測ログ調

### END 2: 家族エンド「ただいま」
- **条件**: `parents` 到達 かつ `bond >= 3` かつ `note` 開封済み
- **概要**: 実家で母が普通に夕飯を作っている。「あんた遅いわよ」隕石の話は誰もしない。

### END 3: 恋人エンド「卒業式」
- **条件**: `school` で同級生と会話 かつ `photo` 所持 かつ `bond >= 2`
- **概要**: 校庭で告白。返事は「は？このタイミングで？」。笑いながら手を繋ぐ。

### END 4: 見知らぬ人エンド「最後のレジ打ち」
- **条件**: コンビニ強盗を止める or ホームレスに `coffee` を渡す + `hope >= 3`
- **概要**: 助けた相手と肩を並べて空を見る。お互い名前を知らない。

### END 5: 達観エンド「無」
- **条件**: `shrine` で瞑想3回 かつ `nihil >= 4`
- **概要**: 神主と並んで座禅。神主「無ですなあ」主人公「無ですね」終わり。

### END 6: 狂気エンド「ヒャッハー」
- **条件**: `wrath >= 4` かつ 略奪フラグ
- **概要**: コンビニのレジから1万円を盗み、神社の賽銭箱を蹴る。最後に空に向かって中指を立てる。**コミカルに描く**。

### END 7: 一人エンド「散歩」 (デフォルト)
- **条件**: 上記いずれにも当てはまらず歩数が0になる
- **概要**: 特に何もせず、近所を歩いていたら終わった。ナレーション「彼は普通の一日を過ごした。最後の一日も。」

## 6. 音声設計

すべて Web Audio API で生成 (外部ファイル不要・即時再生)。

| 音 | 種類 | タイミング |
|----|------|-----------|
| 風音 | ブラウンノイズ + ローパス | BGM (常時、音量 0.05) |
| 鼓動 | 低周波サイン波 (60Hz パルス) | 残り20歩以下、徐々にテンポ↑ |
| クリック音 | 短い矩形波 (1kHz, 30ms) | 選択肢タップ時 |
| 隕石落下 | ホワイトノイズ + ピッチ下降 | エンディング遷移時 |

初回タップで `AudioContext.resume()`。
ヘッダーに **🔇 ミュートボタン** を常設。

## 8. 状態モデル

```js
state = {
  steps: 100,                  // 残り歩数
  currentScene: 'apartment',
  items: ['key_home', 'note'], // ID配列
  params: { bond: 0, nihil: 0, wrath: 0, hope: 0, truth: 0 },
  flags: {},                   // { checkedFridge: true, observedSky: true, ... }
  visited: new Set(),          // 訪問済みシーンID
}

// localStorage
{
  unlockedEndings: ['lonely', 'family'],  // 永続
  lastSave: { ...state },                  // 続きから
}
```

## 9. 実装ステップ (順序)

1. `index.html` + `style.css` 骨組み (モバイルレイアウト + モノクロ)
2. `state.js` + `render.js` で最小ループ (1シーン → 選択 → 別シーン)
3. `scenes.js` に10ノード分のスケルトン定義
4. `ending.js` + `endings.js` で判定 → エンディング画面
5. アイテム・隠しパラメータ反映
6. `audio.js` で環境音 + 効果音
7. 演出 (隕石影、ノイズ、振動)
8. `storage.js` でセーブ + エンド解放履歴 + タイトル画面
9. シーンテキストの肉付け (ブラックユーモア / 真エンドのSF調)
10. GitHub Pages 公開設定確認

## 10. 受け入れ基準

- [ ] スマホ Safari / Chrome で文字が崩れず、タップ精度に問題がない
- [ ] 7エンディングすべて到達可能 (それぞれ最短手順を1つ以上保証)
